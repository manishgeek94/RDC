Please follow these steps to check RDC Digital CMR Services


Check if the services status below appear as running with ibmccmp3 ping :

https://ibmccmp3.portsmouth.uk.ibm.com:9449/CMRServicesV2/cmrs/ping

The following screenshot shows that the services layer is running properly:

If the test returns an error or a blank page, the application may not have been started on the server(IBMCCMP3). A simple check to determine whether the server is running is to execute:

ps -ef|grep cmrservices

It should show like in the below screen :


WebSphere Application Server (WAS) Team from VCS by raising a TSRM/SCCD Ticket can be engaged to help with this issue.

Check if the service layer ping is accessible:

https://ibmccmp3.portsmouth.uk.ibm.com:9445/CISearchServices/service/ping

It should show that the connection to the FindCMR index is successful.


If it's not, check the elastic search index. (use the latest document CI Elastic Loader and Search Services - Operations Guide for instructions).
