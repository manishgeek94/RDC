Follow these steps to check Elastic Search

• All monitoring activities will primarily be done via the Kibana application. To check the health of the cluster, login to Kibana via the URL

http://tpydalespp001.sl.bluecloud.ibm.com:5601/app/kibana

Below is how kibana login page looks like

• Login with your username and password. If you do not have access, please request one from an Elastic Search administrator.

• After logging in, click on Monitoring on the left navigation panel. This will show the overall status of the cluster.

below is how Overall cluster Health looks like

• To check the health of each Elastic Search node, click the Nodes link under Elasticsearch.

below is how ES Nodes Health looks like


• If any node is offline, Login to the server (tpydalespp001.sl.bluecloud.ibm.com) using the mesadmin user. and go to the location /elastic6/elasticsearch-6.2.0/bin. Check first if any instance of Elasticsearch is running by executing the command: ps -ef | grep Elasticsearch

• If anything is down, need to follow the instructions as per the doc (CI Elastic Loader and Search Services - Operations Guide v0.4) from below Box link Box link:

https://ibm.ent.box.com/folder/30280772801
